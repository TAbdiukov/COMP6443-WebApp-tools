The exam will consist of the source of 4 websites being given to you in a tar file. These will mimic the sites you attacked in the break portion of this final exam. Simply identify and fix the security vulnerabilities in each of these sites and submit your diff.

To run these sites you will need python, postgres and in some cases sqlite3 running on your local machine. If you have trouble please use a VPS service, note you may need to install libq-dev on some unix like systems.

Docker

Given the many issues some students had we are defaulting to just python sites with a simple pip install, we will not be asking you to set up a docker image but we will assume you can run bash scripts on your machine

Code


The code which will be provided will have the edited source code of 4 sites you must patch. These sites will not be the exact same as those you saw in the break, they have been cut down and trimmed so only the core components that need to be fixed are present and additional sites have been added. You will most likley have to edit a file called views.py within the structure.

The provided .tar will come with a .git initalized so you can edit and do a git diff at the end to produce a diff file. Note of course that if you git add and commit then all your changes will not be produced in a git diff anymore as they are no longer in staging.
